Vulnerable Query Source

This page reveals the backend code that makes this application vulnerable to PostgreSQL error-based SQL injection.

🔴 Critical Vulnerability

The application uses direct string interpolation to build SQL queries without any parameterization or input validation. When errors occur, verbose PostgreSQL error messages are displayed to the user.

Backend Python Code (Flask)

@app.route('/lookup')
def lookup():
    """VULNERABLE LOOKUP FUNCTION"""
    product_id = request.args.get('id', '')

    # VULNERABLE QUERY: Direct parameter interpolation
    # This allows error-based data extraction
    query = f"SELECT * FROM products WHERE id = {product_id}"

    cursor = conn.cursor()
    cursor.execute(query)
    result = cursor.fetchone()

    return render_template('index.html', product=result)

PostgreSQL Error-Based Injection Techniques

By crafting specific inputs that cause PostgreSQL errors, you can extract sensitive data:

  • Type Casting: Use CAST(expr AS type) or expr::type syntax to force type conversions that reveal data
  • Column Enumeration: Trigger errors that list column names and types in error messages
  • Table Discovery: Use pg_tables and information_schema with type casting to find hidden tables
  • Data Extraction: Extract data character-by-character using conditional type casting errors
  • Syntax Leakage: Malformed queries reveal the underlying SQL structure
